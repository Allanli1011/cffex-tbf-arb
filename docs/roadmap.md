# 分阶段开发计划

> 每个 Phase 结束后做一次回顾与文档更新。

## Phase 0 — 文档与脚手架（当前）

**目标**：完成项目结构、设计文档、依赖管理。

- [x] 创建项目目录与 git 仓库
- [x] 撰写 design.md / strategies.md / roadmap.md
- [ ] 准备 Python 环境（pyproject.toml / requirements.txt）
- [ ] 初始化 configs（合约元数据、CF 表占位、阈值参数）
- [ ] 加 .gitignore、首个 commit

**交付物**：可被新人 clone 后理解整体目标的代码库骨架。

## Phase 1 — 数据层

**目标**：稳定的行情与基础数据接入。

- [ ] 合约元数据表（contract_id, product, listing_date, last_trade_date, CF）
- [ ] 可交割券池维护脚本
- [ ] 期货行情接入（先用日频公开数据，后续接 CTP / CFFEX L2）
- [ ] 现券收益率与价格接入（初期可用 Wind / iFinD 导出 CSV）
- [ ] 回购利率接入（DR007 / GC007 / FR007）
- [ ] 数据校验与缺失处理

**交付物**：能拉取过去 1 年日频数据并落库的 ETL 脚本。

## Phase 2 — 定价与信号引擎

**目标**：基差、IRR、CTD、跨期价差、曲线指标的计算。

- [ ] 应计利息与转换因子计算（验证与 CFFEX 公布值一致）
- [ ] Gross Basis / Net Basis / IRR 计算器
- [ ] CTD 识别与切换概率
- [ ] 跨期价差计算与历史分位数
- [ ] 期货隐含 yield 与 DV01 计算（先用久期近似，再升级 QuantLib OAS）
- [ ] 蝶式 / 陡平价差计算

**交付物**：可对历史数据生成全量信号时序的引擎，单元测试覆盖关键公式。

## Phase 3 — 回测框架

**目标**：策略历史表现评估。

- [ ] 事件驱动回测引擎（处理保证金、滑点、手续费）
- [ ] 基差套利回测
- [ ] 跨期套利回测
- [ ] 蝶式 / 陡平回测
- [ ] 绩效指标：PnL / Sharpe / MaxDD / 胜率
- [ ] 参数扫描与敏感度分析

**交付物**：每个策略的回测报告（notebook + 图表）。

## Phase 4 — 监控面板（MVP）

**目标**：Web 端实时展示。

- [ ] 后端 API（FastAPI）：合约、价格、信号、历史数据
- [ ] 前端框架（React + ECharts）
- [ ] 模块 A 总览驾驶舱
- [ ] 模块 B 基差监控
- [ ] WebSocket 推送实时信号

**交付物**：本机运行的可视化面板，覆盖最关键的两个模块。

## Phase 5 — 监控面板（完整版）

- [ ] 模块 C 跨期监控
- [ ] 模块 D 曲线套利监控
- [ ] 模块 E CTD 与交割分析
- [ ] 模块 F 回测与统计
- [ ] 模块 G 风险与执行
- [ ] 模块 H 信号告警与推送

## Phase 6 — 增强

- [ ] 机器学习信号层
- [ ] Regime Detection

- [ ] 压力测试场景库

## Phase 7 — 实盘联调（可选）

- [ ] CTP / 柜台 API 接入（仅查询）
- [ ] 模拟单与算法执行器
- [ ] 风控前置（DV01 限额、品种集中度）

## 优先级建议

如果资源有限，按以下优先级推进：

1. **Phase 0 → Phase 1 → Phase 2** 是地基，必须完成
2. **基差监控** 价值密度最高，建议作为 Phase 4 的第一个面板
3. **曲线套利** 需要 4 品种数据完整，可放后期
4. **回测框架** 与定价引擎并行开发，互相验证
