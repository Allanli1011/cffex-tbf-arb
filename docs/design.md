# 中金所国债期货套利策略监控面板 — 总体设计方案

> 版本：v0.2（2026-04-26）
> 状态：草案
> 关键约束：**仅使用开源免费数据，日频分析**

## 一、产品基础与套利标的

中金所国债期货共 4 个品种，建议监控全谱系：

| 品种 | 久期 | 合约规模（面值） | 主要用途 |
|------|------|------------------|---------|
| TS（2 年） | ~1.9 | 200 万 | 短端、跨品种 |
| TF（5 年） | ~4.5 | 100 万 | 蝶式中腿 |
| T（10 年） | ~7.8 | 100 万 | 主力、流动性最佳 |
| TL（30 年） | ~18 | 100 万 | 超长端、曲线交易 |

每品种 4 个连续季月合约同时挂牌（例如 TS2503 / 2506 / 2509 / 2512），合计同时跟踪 16 个合约 + 数十只可交割券。

## 二、核心套利策略矩阵

### 1. 基差套利（Basis Trading）

**核心公式**

```
Gross Basis = P_bond - F × CF
Net Basis  = Gross Basis - Carry            # Carry = 票息收入 - 资金成本
IRR        = [(F × CF + AI_T) / (P_bond + AI_0) - 1] × 365 / n
```

**监控指标**
- 每只可交割券的 Gross Basis / Net Basis / IRR / BNOC（Basis Net of Carry）
- IRR vs R007 / GC007 / FR007 利差。信号阈值参考：> 30bp 开正套，< -20bp 开反套
- CTD 切换概率（基于久期偏离与收益率变动模拟）
- 交割期权价值（switch / timing / wild card）

**典型策略**
- 正向基差：买现券 + 卖期货（IRR > 资金成本）
- 反向基差：融券卖出 + 买期货（IRR < 0 或显著低于回购，国内融券难度大）
- CTD 切换博弈：在切换临界点持有可能成为新 CTD 的券

### 2. 跨期套利（Calendar Spread）

**理论价差**

```
F_far - F_near ≈ F_near × (r - c_implied) × Δt / 365
```

**监控指标**
- 近-次主力价差及其历史分位数（1Y / 3Y rolling percentile）
- 移仓窗口的滚动收益（roll yield）
- 远月 vs 近月隐含融资利差
- 持仓量迁移曲线（识别移仓节奏）

**典型策略**
- 价差走阔到 95 分位：卖远买近
- 移仓套利：在多头集中移仓前做空远月、做多近月
- 跨期 + 现券组合：构造完整的 carry trade

### 3. 收益率曲线套利（Curve Arbitrage）

**核心组合（DV01 中性）**

| 策略 | 头寸 | 观察利差 |
|------|------|---------|
| 2s10s steepener | +TS, -T | 看陡 |
| 5s30s flattener | -TF, +TL | 看平 |
| 2-5-10 蝶式 | +TS, -2×TF, +T（按 DV01 加权） | 5Y 相对凸性 |
| 5-10-30 蝶式 | +TF, -2×T, +TL | 10Y 相对凸性 |

**监控指标**
- 期货隐含 yield 的曲线斜率与曲率
- 期货曲线 vs 现券曲线的背离度（识别期货端 mispricing）
- DV01 中性、Convexity 中性两套权重并行展示
- Z-score 与历史分位数

### 4. 其他可拓展套利场景

| 策略 | 思路 | 难点 |
|------|------|------|
| 期货 vs IRS | 期货隐含收益率 vs FR007 / Shibor3M IRS | 基差风险、信用利差 |
| 国债期货 vs 国开债 | 国开-国债利差期货化对冲 | 国开无对应期货，需用现券 |
| 跨市场套利 | 银行间 vs 交易所同券价差 | 跨市场资金效率 |
| CTD 池内换券 | 同合约不同 CTD 候选间套利 | 流动性、回购可得性 |
| 期货+回购组合 | 用期货合成短期资金利率，与实际 repo 比较 | 类似 IRR 套利 |
| 指数增强对冲 | 债基贝塔对冲 | 久期匹配 |
| 跨境套利（远期） | 中美利差 + 货币掉期 vs 国内期货 | 资本管制 |

## 三、监控面板模块设计

> **日频约束下的统一调整**：所有"实时"展示在 MVP 中均为"**日终批处理 + 静态刷新**"。监控面板每日收盘后约 1–2 小时（待 CFFEX 结算与中债估值发布完成）刷新一次。

### A — 总览驾驶舱
- 4 品种主力合约日终价格、基差、IRR 一屏热力图
- 当日机会信号 Top10（按 Z-score 排序）
- 资金面摘要：DR007、GC007、FR007 收盘
- 风险预警灯（持仓集中度、CTD 切换临界、移仓窗口）

### B — 基差监控
- 每个合约可交割券明细（券代码、CF、收盘净价、Gross / Net Basis、IRR、BNOC、剩余天数）
- IRR 日频时序图 + 资金利率叠加
- CTD 轨迹图（哪只券在哪段时间为 CTD）
- 交割期权价值估算（蒙特卡洛或 Black 简化）

### C — 跨期监控
- 近-次价差日 K 线 + 分位数带（5 / 25 / 50 / 75 / 95）
- 持仓迁移热图（基于中金所每日持仓排名）
- 移仓成本计算器（输入持仓量，输出滑点 + 价差成本）

### D — 曲线套利监控
- DV01 中性比例计算器（日终估值）
- 蝶式价差时序、Z-score、回测累计 PnL
- 期货隐含曲线 vs 中债收益率曲线对照图
- 凸性调整后的"理论 fair spread"

### E — CTD 与交割分析
- 所有可交割券的 yield-CF 散点
- 收益率±50bp 情景下的 CTD 切换矩阵
- 隐含期权价值分解（switch / timing / wild card）

### F — 回测与统计
- 任一策略历史 PnL、Sharpe、最大回撤、胜率
- 入场 / 出场阈值参数化扫描热图
- 滑点 / 手续费敏感度分析

### G — 风险与持仓分析
- 持仓 DV01、Key Rate Duration（2 / 5 / 10 / 30）
- VaR / Expected Shortfall（历史模拟法）
- 信号触发记录与策略组合敞口
- *（实盘算法执行模块在开源数据约束下不在范围内）*

### H — 信号与告警
- 阈值触发：IRR、价差分位数、CTD 切换概率
- 推送通道：邮件 / 钉钉 webhook / 企微 webhook（日终批量）
- 信号回看与命中率统计

## 四、数据与技术架构

### 数据源（开源免费、日频）

**核心工具：[AKShare](https://github.com/akfamily/akshare)** — MIT 协议的开源金融数据接口，覆盖本项目 80% 的需求。

| 数据类别 | 来源 | 接入方式 | 频率 |
|---------|------|---------|------|
| 国债期货日线（TS/TF/T/TL） | 新浪财经 / 东方财富 | `akshare.futures_zh_daily_sina` | 日频 |
| 期货持仓排名 | 中金所官网 | `akshare.get_cffex_rank_table` | 日频 |
| 可交割券池 | 中金所公告 | 自写爬虫（`http://www.cffex.com.cn/jystz/`） | 季度 + 增量 |
| **转换因子（CF）** | 中金所新合约挂牌公告 | 自写爬虫 + append-only 存储 | 见下文 |
| 中债收益率曲线 | 中国债券信息网 | `akshare.bond_china_yield` | 日频 |
| 国债现券估值 | 中债估值 / 财政部 | `akshare.bond_zh_us_rate` 等 | 日频 |
| 银行间回购 DR007 / R007 | 中国货币网 | `akshare.repo_rate_hist` | 日频 |
| 交易所回购 GC007 | 上交所 | AKShare 日频接口 | 日频 |
| Shibor / FR007 | 上海清算所 / 货币网 | `akshare.macro_china_shibor_all` | 日频 |
| 交易日历 | 中金所 / 新浪 | `akshare.tool_trade_date_hist_sina` | 静态 |

**兜底数据源**：CFFEX 官网直爬（每日交易数据 / 持仓 / 公告 PDF），作为 AKShare 不可用时的备份与权威源。

### 数据约束与降级说明

| 数据 | 限制 | 系统层面影响 |
|------|------|-------------|
| Tick / Level-2 行情 | 商业授权才能获取 | 实时套利信号无法做；日频策略不受影响 |
| 银行间现券逐笔成交 | CFETS 会员才能拿 | 流动性评分降级为 "收盘价 + 日成交量" 近似 |
| 实时回购利率盘口 | 同上 | 用日频 R007 / DR007 收盘代替 |
| 期货实时行情订阅 | CTP 接口需开户 | 监控面板从"实时推送"降级为"**日终批量更新**" |
| 算法执行 / 实盘下单 | 需期货账户 + CTP 接入 | **不在 MVP 范围**，仅做信号与回测 |

### 转换因子（CF）维护：append-only

CF 一旦由中金所公告即**永久不变**。维护策略：

- **写入即冻结**：`(合约代码, 券代码) → CF` 是一对永久键值，脚本在追加时若发现冲突值必须报错而不是覆盖
- **季度全量挂牌**：每季度（3/6/9/12 月末）4 个品种各挂牌 1 个新合约，需拉取约 4 张新 CF 表
- **增量补券**：合约存续期内若财政部新发券进入可交割池，中金所会就该券补发 CF 公告，需追加新行
- **唯一可信源**：中金所交易所公告页 `http://www.cffex.com.cn/jystz/` 的合约挂牌通知与可交割券补充通知

存储采用 `configs/cf_table.csv`（或 SQLite 表），脚本 `scripts/refresh_cf.py` 季度自动运行 + 公告增量手动触发。

### 技术栈（精简版）

```
数据存储:  SQLite + Parquet 文件
           - SQLite: 合约元数据、CF 表、信号记录、回测结果（结构化、需要事务）
           - Parquet: 期货日线、现券估值、收益率曲线（时序、列存高效）
计算层:    Python + pandas + numpy + QuantLib (债券定价 / DV01)
任务调度:  Cron / Python schedule（日终批处理）
监控面板:  Streamlit（MVP）→ 升级路径：FastAPI + 静态 HTML
告警通道:  邮件 SMTP / 钉钉 / 企微 webhook
```

**省略的组件**（vs 原方案）：ClickHouse、Redis、PostgreSQL、WebSocket、React、Airflow。日频 + 数据量小（一年约几十万行）的场景下完全不需要。

### 关键工程点
1. **CF 表 append-only**：脚本必须做冲突检测，已写入的 (合约, 券) 对永远不能改值
2. **应计利息计算**：使用 ACT/ACT 或 30/360 日计基准，需与中债估值口径一致
3. **收益率插值**：从中债曲线节点（1Y/3Y/5Y/7Y/10Y/30Y 等）插值出任意期限——先用 Hermite，后续可升级 Nelson-Siegel-Svensson
4. **节假日与交易日历**：银行间、交易所、期货市场各不相同，统一用 `akshare.tool_trade_date_hist_sina` 兜底
5. **DV01 计算**：先用修正久期近似，后续用 QuantLib 走完整定价（特别是 TL 的凸性影响）
6. **数据校验**：每个 ETL 任务结束后做 sanity check（价格不为 0、日期连续、CF 在合理范围 0.85–1.20 等）

## 五、可选增强方向

- **机器学习信号层**：用 LightGBM / Transformer 预测基差均值回归概率
- **Regime Detection**：识别牛陡 / 熊陡 / 牛平 / 熊平四种环境，分场景调参
- **流动性评分**：结合盘口深度、成交量给可交割券打分，避开"看似便宜实则做不进"的券
- **压力测试**：2016 年钱荒、2020 年永煤、2022 年理财赎回等历史情景的策略表现回放

## 六、风险与免责说明

本系统用于研究与监控目的，所有信号仅作辅助决策。实盘交易需结合：
- 流动性约束（盘口深度、冲击成本）
- 监管限制（持仓限额、自成交规则）
- 资金成本与融券可得性
- 交割流程与税收影响
