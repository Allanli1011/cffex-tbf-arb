# 中金所国债期货套利策略监控面板 — 总体设计方案

> 版本：v0.1（2026-04-26）
> 状态：草案

## 一、产品基础与套利标的

中金所国债期货共 4 个品种，建议监控全谱系：

| 品种 | 久期 | 合约规模（面值） | 主要用途 |
|------|------|------------------|---------|
| TS（2 年） | ~1.9 | 200 万 | 短端、跨品种 |
| TF（5 年） | ~4.5 | 100 万 | 蝶式中腿 |
| T（10 年） | ~7.8 | 100 万 | 主力、流动性最佳 |
| TL（30 年） | ~18 | 100 万 | 超长端、曲线交易 |

每品种 4 个连续季月合约同时挂牌（例如 TS2503 / 2506 / 2509 / 2512），合计同时跟踪 16 个合约 + 数十只可交割券。

## 二、核心套利策略矩阵

### 1. 基差套利（Basis Trading）

**核心公式**

```
Gross Basis = P_bond - F × CF
Net Basis  = Gross Basis - Carry            # Carry = 票息收入 - 资金成本
IRR        = [(F × CF + AI_T) / (P_bond + AI_0) - 1] × 365 / n
```

**监控指标**
- 每只可交割券的 Gross Basis / Net Basis / IRR / BNOC（Basis Net of Carry）
- IRR vs R007 / GC007 / FR007 利差。信号阈值参考：> 30bp 开正套，< -20bp 开反套
- CTD 切换概率（基于久期偏离与收益率变动模拟）
- 交割期权价值（switch / timing / wild card）

**典型策略**
- 正向基差：买现券 + 卖期货（IRR > 资金成本）
- 反向基差：融券卖出 + 买期货（IRR < 0 或显著低于回购，国内融券难度大）
- CTD 切换博弈：在切换临界点持有可能成为新 CTD 的券

### 2. 跨期套利（Calendar Spread）

**理论价差**

```
F_far - F_near ≈ F_near × (r - c_implied) × Δt / 365
```

**监控指标**
- 近-次主力价差及其历史分位数（1Y / 3Y rolling percentile）
- 移仓窗口的滚动收益（roll yield）
- 远月 vs 近月隐含融资利差
- 持仓量迁移曲线（识别移仓节奏）

**典型策略**
- 价差走阔到 95 分位：卖远买近
- 移仓套利：在多头集中移仓前做空远月、做多近月
- 跨期 + 现券组合：构造完整的 carry trade

### 3. 收益率曲线套利（Curve Arbitrage）

**核心组合（DV01 中性）**

| 策略 | 头寸 | 观察利差 |
|------|------|---------|
| 2s10s steepener | +TS, -T | 看陡 |
| 5s30s flattener | -TF, +TL | 看平 |
| 2-5-10 蝶式 | +TS, -2×TF, +T（按 DV01 加权） | 5Y 相对凸性 |
| 5-10-30 蝶式 | +TF, -2×T, +TL | 10Y 相对凸性 |

**监控指标**
- 期货隐含 yield 的曲线斜率与曲率
- 期货曲线 vs 现券曲线的背离度（识别期货端 mispricing）
- DV01 中性、Convexity 中性两套权重并行展示
- Z-score 与历史分位数

### 4. 其他可拓展套利场景

| 策略 | 思路 | 难点 |
|------|------|------|
| 期货 vs IRS | 期货隐含收益率 vs FR007 / Shibor3M IRS | 基差风险、信用利差 |
| 国债期货 vs 国开债 | 国开-国债利差期货化对冲 | 国开无对应期货，需用现券 |
| 跨市场套利 | 银行间 vs 交易所同券价差 | 跨市场资金效率 |
| CTD 池内换券 | 同合约不同 CTD 候选间套利 | 流动性、回购可得性 |
| 期货+回购组合 | 用期货合成短期资金利率，与实际 repo 比较 | 类似 IRR 套利 |
| 指数增强对冲 | 债基贝塔对冲 | 久期匹配 |
| 跨境套利（远期） | 中美利差 + 货币掉期 vs 国内期货 | 资本管制 |

## 三、监控面板模块设计

### A — 总览驾驶舱
- 4 品种主力合约价格、基差、IRR 一屏热力图
- 当日机会信号 Top10（按 Z-score 排序）
- 资金面摘要：DR007、GC007、回购曲线
- 风险预警灯（持仓集中度、CTD 切换临界、移仓窗口）

### B — 基差监控
- 每个合约可交割券明细（券代码、CF、价格、Gross / Net Basis、IRR、BNOC、剩余天数）
- IRR 时序图 + 资金利率叠加
- CTD 轨迹图（哪只券在哪段时间为 CTD）
- 交割期权价值估算（蒙特卡洛或 Black 简化）

### C — 跨期监控
- 近-次价差 K 线 + 分位数带（5 / 25 / 50 / 75 / 95）
- 持仓迁移热图
- 移仓成本计算器（输入持仓量，输出滑点 + 价差成本）

### D — 曲线套利监控
- 实时 DV01 中性比例计算器
- 蝶式价差时序、Z-score、回测累计 PnL
- 期货隐含曲线 vs 现券曲线对照图
- 凸性调整后的"理论 fair spread"

### E — CTD 与交割分析
- 所有可交割券的 yield-CF 散点
- 收益率±50bp 情景下的 CTD 切换矩阵
- 隐含期权价值分解（switch / timing / wild card）

### F — 回测与统计
- 任一策略历史 PnL、Sharpe、最大回撤、胜率
- 入场 / 出场阈值参数化扫描热图
- 滑点 / 手续费敏感度分析

### G — 风险与执行
- 实时持仓 DV01、Key Rate Duration（2 / 5 / 10 / 30）
- VaR / Expected Shortfall（历史模拟法）
- 交易日志 + 算法执行进度（TWAP / VWAP 拆单）

### H — 信号与告警
- 阈值触发：IRR、价差分位数、CTD 切换概率
- 推送通道：钉钉 / 企微 / 邮件
- 信号回看与命中率统计

## 四、数据与技术架构

### 数据源
- 行情：CTP / CFFEX Level-2、CFETS（现券）、Wind / iFinD（兜底）
- 回购：CFETS（DR / R 系列）、上交所（GC 系列）
- 基础信息：可交割券池、CF 表、应计利息

### 技术栈

```
数据层：   ClickHouse (tick / 分钟) + PostgreSQL (合约元数据 / CF / 持仓)
计算层：   Python (QuantLib + 自研定价引擎) + Redis (实时缓存)
推送层：   WebSocket / Server-Sent Events
前端：     React + ECharts / TradingView Lightweight Charts
任务调度： Airflow / Prefect (日终结算、CTD 重算、回测)
```

### 关键工程点
1. CF 表自动更新（每年 12 月新合约挂牌时）
2. 收益率插值：Hermite / Nelson-Siegel-Svensson 双引擎
3. 节假日与交易日历（大商所 / 上交所 / 银行间各不相同）
4. 实时 DV01 用 OAS 法而非简单久期，TL 要考虑凸性

## 五、可选增强方向

- **机器学习信号层**：用 LightGBM / Transformer 预测基差均值回归概率
- **Regime Detection**：识别牛陡 / 熊陡 / 牛平 / 熊平四种环境，分场景调参
- **流动性评分**：结合盘口深度、成交量给可交割券打分，避开"看似便宜实则做不进"的券
- **压力测试**：2016 年钱荒、2020 年永煤、2022 年理财赎回等历史情景的策略表现回放

## 六、风险与免责说明

本系统用于研究与监控目的，所有信号仅作辅助决策。实盘交易需结合：
- 流动性约束（盘口深度、冲击成本）
- 监管限制（持仓限额、自成交规则）
- 资金成本与融券可得性
- 交割流程与税收影响
